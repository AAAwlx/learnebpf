## 第二部分：基础监控功能（3分钟）

### 场景4：直接运行 bpftrace（2:00-3:00）

**画面**：

**旁白**：
> "首先，让我们用最简单的方式开始 - 直接运行 bpftrace 脚本。
> 这是 bpftrace 版本，无需编译，可以直接运行。"

**操作**：
```bash
# 终端1：启动监控
sudo ./netfilter_trace.bt
```

**旁白**：
> "现在监控已经启动了。让我们在另一个终端触发一些网络流量。"

**操作**：
```bash
# 终端2：触发流量
ping -c 3 8.8.8.8
curl https://www.google.com > /dev/null
```

**旁白**：
> "你可以看到，每一条网络请求都产生了大量的跟踪信息。
> 我们可以看到 [ip_rcv] 表示接收，[ip_local_deliver] 表示本地投递，
> [nf_hook_slow] 显示了 netfilter 的钩子处理，
> 最后 [ACCEPT] 表示这个包被接受了。"

**操作**：
- 按 Ctrl+C 停止监控

---

### 场景5：理解输出格式（3:00-4:00）

**画面**：
- 保存刚才的输出到文件
- 展示并讲解

**旁白**：
> "让我们详细分析一下输出格式。

> 第一行显示函数名，比如 ip_rcv，
> 然后是源 IP 和目的 IP，最后是 skb 的内存地址。

> nf_hook_slow 这一行特别重要，它显示了 hook_num，
> 这表示数据包通过了哪个 netfilter 钩子点。
> 0 表示 PRE_ROUTING，1 表示 LOCAL_IN，等等。

> 最后，kretprobe 显示了 netfilter 的最终判断：
> ACCEPT 表示接受，DROP 表示丢弃。"

**操作**：
```bash
# 在终端展示示例输出
cat << 'EOF'
[ip_rcv] 192.168.1.100 -> 8.8.8.8 | skb:0xffff888123456789
[nf_hook_slow] 192.168.1.100 -> 8.8.8.8 | skb:0xffff... | hook_num:0
[ACCEPT] 192.168.1.100 -> 8.8.8.8 | skb=0xffff888123456789
EOF
```
